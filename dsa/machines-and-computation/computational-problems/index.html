Computational problems

A computational problem is specified by:

  • The name of the problem.
  • The inputs and their representation.
  • The outputs for a given input and their representation.
  • The effects of the computation on the machine for a given input.

The primary school computational problem of adding two numbers can be specified as follows:

Name
ADD
Input
Two numbers \(n\) and \(m\) in decimal.
Output
The sum \(n+m\) in decimal.
Effect
None.

The numbers are presented to you in decimal representation. The output of the problem is related to inputs via the equation \(x = n+m\) and \(x\) is expected to be in decimal representation. This problem statement is device independent. i.e., the problem may be reasonably expected to be solved by a human (which is typically the case in primary school) or a computer.

When specifying problems in this text, we will assume that it is to be solved by a specific computational device, like a digital computer. Therefore, we may incorporate some details of the machine into the problem specification. For example, the sorting problem, which is solved by Python’s list.sort() or C’s qsort() functions can be specified as:

Name
SORT
Input
An array a of n elements along with an ordering relation on the elements.
Output
None.
Effect
The elements of the array are reordered into ascending order.

Notice that the resulting sequence is obtained as an effect of the computation. i.e., the storage holding the input array is modified so that it becomes the desired output array. The problem specification requires no output. So these functions return no values.

A variant of SORT where there are no effects where the input sequence remains the same and the output is the sorted sequence may also be defined. The following Python function solves this variant of SORT.

def sort(xs):
      return list(sorted(xs))
