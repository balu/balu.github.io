Computational problems

We use computing to solve computational problems. A computational problem should be specified by a (1) name, (2) description of its inputs, and (3) description of its outputs including how the inputs and outputs are related. The problem of sorting a sequence of elements can be specified as:

Name
SORT
Input
A sequence of elements and an ordering relation among the elements.
Output
The same sequence of elements in ascending order.

We can also make the specification more precise by using mathematical notation.

Name
SORT
Input
  • A sequence \(S = (a_0, \dotsc, a_{n-1})\) where each \(a_i\in A\) for some set \(A\).
  • A total order \(\leq\) on the set \(A\).
Output
A sequence \(T = (b_0, \dotsc, b_{n-1})\) that satisfies:
  • The sequence \(T\) can be obtained from \(S\) by only rearranging the elements of \(S\).
  • \(b_0 \leq b_1 \dotsc \leq b_{n-1}\).

As with all writing, the specification you use should depend on the audience. The simplest specification that conveys the full information is the best one.

Exercise

  1. Define the computational problem of subtracting two natural numbers.
  2. Define the computational problem of computing square roots.
  3. Define the computational problem of finding the largest element in a sequence of natural numbers.
  4. Define the computational problem of finding an element in a sequence.
  5. Show how to mathematically specify: “The sequence \(T\) can be obtained from \(S\) by only rearranging the elements of \(S\)”.
